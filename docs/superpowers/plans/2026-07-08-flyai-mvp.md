# FlyAI MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the main hotel competitor monitor flow with a minimal FlyAI/Fliggy data pipeline, WorkBuddy analysis prompt, and ClawBot push path.

**Architecture:** The main flow is API-first: a PowerShell script reads a local JSON config, calls the FlyAI CLI, saves raw FlyAI output under `data/`, and creates a report input Markdown file. WorkBuddy reads that input, uses the built-in model with `templates/daily-prompt.md`, saves a report under `reports/`, and sends the final report through ClawBot.

**Tech Stack:** Static HTML/JS local wizard, Windows PowerShell, FlyAI CLI, WorkBuddy built-in model, ClawBot, optional enterprise WeChat webhook fallback.
