# Hotel Competitor Monitor v0.2 Reliability and Data Truth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent stale or incorrectly scoped hotel reports by making failures propagate, applying supported FlyAI query fields, validating hotel identity, treating masked prices as non-numeric, persisting Baidu daily usage, and making the local wizard honor current form and push settings.

**Architecture:** Keep the existing PowerShell orchestration, but extract deterministic matching, price, bed-type, and daily-usage functions into a small reusable module. The collector remains the owner of API sequencing; `run-once.ps1` becomes the trustworthy process boundary; the frontend remains dependency-free and generates config/prompt files locally.

**Tech Stack:** Windows PowerShell 5.1, Node.js 20+ test runner, vanilla HTML/CSS/JavaScript, local JSON/Markdown artifacts.
