# 数据源说明

## MVP 选择

本项目 MVP 选择 FlyAI/飞猪作为主数据源，原因是：

- 可以按城市、商圈/POI、关键词和日期搜索酒店。
- 带 Key 后可以返回飞猪生态下的价格信号。
- 返回酒店名称、地址、档位、品牌、经纬度等字段时，适合做竞品候选和分层。
- 比浏览器读取网页更快、更稳定，也更容易自动化。

## 数据限制

- FlyAI/飞猪价格不代表全网最低价。
- 如果返回 `¥1xx`、`¥3x` 等脱敏价格，只能做价格带判断。
- 评分、评论、早餐、取消政策等字段不是 MVP 主目标。
- 核心竞品最终仍建议人工复核。

## 分层规则

- 核心竞品：距离近、档位接近、价格重叠、品牌客群接近。
- 价格压力：距离近但价格明显更低。
- 延展竞品：距离稍远但客源可能重叠。
- 替代竞品：公寓、民宿、电竞酒店、青年旅舍等非标住宿。
- 剔除：距离远、档位差异过大、业态不相关或数据不足。
