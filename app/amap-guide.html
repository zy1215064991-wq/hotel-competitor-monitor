配置顺序

高德负责本店定位和周边酒店候选池。项目会使用文本检索和周边检索，不负责价格和评论。

1

创建 Web 服务 Key

打开官方入口，创建应用并选择 Web 服务类型 Key。

https://lbs.amap.com/api/webservice/create-project-and-key
2

确认服务权限

确保 Key 可用于地点文本检索和周边检索。后续脚本会按城市、本店名称、POI 和半径生成候选池。

3

写入 Windows 环境变量

setx AMAP_API_KEY "替换成你的高德 Key"
4

本地自检

if ($env:AMAP_API_KEY) { "AMAP_API_KEY 已配置" } else { "缺少 AMAP_API_KEY" }

不要用 echo 打印 Key 明文。配置后重新打开 WorkBuddy 或 PowerShell。

运行边界

DryRun 不调用高德。正式运行会先定位本店，再按半径获取候选池，结果写入 data/api-combo/。
